# Security Policy

We take the security of this project seriously. Please follow the guidance below to report vulnerabilities or security concerns.

## Reporting a Vulnerability

Please do NOT create public issues that include sensitive details.

Preferred reporting methods on GitHub:

1. Private vulnerability report (if enabled)
   - Go to the repository's Security tab and use "Report a vulnerability" to submit a private report to the maintainers.
   - URL: https://github.com/6NineLives/AI-Essentials/security

2. If private reporting is not available
   - Open a brief issue titled `Security: request private contact` with no exploit details.
   - A maintainer will follow up with a private channel.
   - Issues: https://github.com/6NineLives/AI-Essentials/issues/new

## What to include (privately)

- A clear description of the issue and potential impact
- Steps to reproduce (PoC) and affected files/paths
- Environment details (OS, versions, commit SHA)
- Any temporary mitigations or workarounds

## Our commitment

- We will acknowledge your report within 72 hours
- We will work to validate the issue quickly and keep you updated
- We will coordinate disclosure and remediation steps responsibly

## Scope

This policy covers code and configurations in this repository. Security issues in third-party dependencies should be reported to their respective projects.

## Safe harbor

We will not initiate legal action for good-faith, non-destructive research that follows this policy and respects user privacy and data protection.
